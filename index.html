Plug-and-play
architecture.

Nexus separates your raw data from the twin model. Upload once, map your schema, and the simulation engine handles the rest. No engineering team required.

✓
Schema mapping — auto-detect columns, map to Nexus entities (SKU, supplier, warehouse), and validate before import.
✓
Incremental sync — push deltas via API or schedule full refreshes. The twin stays current without manual re-uploads.
✓
Data sandbox — test new data against the existing twin before promoting to production. No risk to live dashboards.
// Example: Upload inventory data
POST /api/v1/data/inventory
{
  "source": "erp_export.xlsx",
  "mapping": {
    "sku_col": "Item Code",
    "qty_col": "On Hand",
    "warehouse_col": "Location"
  }
}
