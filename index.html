Data-first.
Replace mock JSON with
your real operations.

The simulation engine is decoupled from the UI. Swap the data layer — the twin rebuilds itself. No redesign, no re-engineering.

✓
Schema mapping — auto-detect columns, map to Nexus entities, validate before import
✓
Incremental sync — push deltas via API or schedule full refreshes
✓
Data sandbox — test new data against the existing twin before going live

// Simulation Engine

Model chaos
before it arrives.

Adjust four parameters — demand volume, supplier delay, labor availability, and reorder policy — and watch every KPI recalculate downstream in real time.

✓
Fill rate impact — see exactly which customer regions degrade first
✓
Cost modeling — carrying cost shifts dynamically with inventory levels
✓
Stockout prediction — identify at-risk SKUs before they hit zero
// Scenario: Port congestion + peak season engine.setParams({ demand: 160, // +60% holiday spike delay: 14, // 2-week port backup labor: 75, // seasonal staffing reorder: 3 // aggressive restocking }); // Result: fill rate drops to 82.4% // 5 SKUs enter stockout risk zone // carrying cost spikes +41%
